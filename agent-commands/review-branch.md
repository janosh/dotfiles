# Review Current Branch

## Goal

Perform a comprehensive code review of the current feature branch before merging to main.

## Steps

1. **Review:** Analyze the diff between the current branch and `main`.
2. **Identify:** Look for:
    - Potential bugs
    - Performance bottlenecks
    - Code smells / Verbosity
3. **Rank:** Prioritize findings.
4. **Fix:**
    - If confident: Implement the fixes immediately.
    - If unsure: Ask clarifying questions (bundle them at the end).

## Outcome

Make the codebase robust, readable, and concise.
